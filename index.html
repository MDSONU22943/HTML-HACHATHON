How Does the Internet Work?

Principle 1: Sharing Information

Imagine you have a file on your computer and want to send it to your friend's computer far away. Instead of carrying a USB physically, we use the internet to transfer data electronically.

Principle 2: The Problem of Many Computers

Connecting every computer to every other computer directly is not practical. Instead, we use a central device called a Switch to connect many computers in one place (Local Network).

Principle 3: Connecting Networks

To connect different networks (like your home and your friend's office), we use a Router. Routers forward messages between networks.

Principle 4: Finding Devices (IP Address)

Every computer has a unique IP Address like a home address, so data knows where to go.

Principle 5: Reliability (Packets)

Large files are broken into small pieces called Packets. Each packet carries part of the file, sender & receiver address, and sequence number so they can be reassembled properly.

Summary

  • Direct Link: Share data between two computers.
  • Switch: Connect many computers locally.
  • Router: Connect different networks.
  • IP Address: Unique address for every device.
  • Packets & Protocols: Reliable data transfer using TCP/IP.

What is an IP Address?

An IP Address is like your house address, but for your computer on the Internet. It helps other computers know where to send data.

Example IPv4 Address: 192.168.1.1

Example IPv6 Address: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

Simple Example: If you want to send a letter to your friend, you need their house address. Similarly, data on the internet is sent using IP Addresses.

Public vs Private IP

Public IP: The address visible on the internet, like your home's address where the world can reach you.

Private IP: Address used within your home or office network, like the room number inside your home.

What is a MAC Address?

A MAC Address is a unique serial number given to every network device (like your computer’s Wi-Fi card).

Example MAC Address: 3C:22:FB:A3:B4:C5

Simple Explanation: Think of it like a fingerprint – no two devices have the same MAC Address. It helps devices in your local network recognize each other.

How to Find MAC Address:

  • Windows: Open Command Prompt and type ipconfig /all or getmac /v.
  • macOS: Open Terminal and type ifconfig.

What is a Port Number?

A Port Number is like the door number of a house. It helps computers know which application should receive the data.

Example:

  • HTTP (Web Page): Port 80
  • HTTPS (Secure Web): Port 443

Simple Analogy: Imagine your house has many doors. Port Numbers tell data through which door it should go to reach the correct service (like web browser, email, database).

Port Categories:

  1. Well-Known Ports (0–1023): Standard apps (HTTP, FTP).
  2. Registered Ports (1024–49151): Specific apps (MySQL, MongoDB).
  3. Dynamic Ports (49152–65535): Temporary use by programs.

What is DNS (Domain Name System)?

DNS is like the phonebook of the internet. Instead of remembering IP Addresses, you type google.com, and DNS finds the correct IP address for you.

How DNS works (Step by Step):

  1. Local Cache: Your computer first checks if it already knows the answer.
  2. Recursive Resolver: Asks ISP's DNS Server (like Google's 8.8.8.8).
  3. Hierarchy Search: Root → TLD → Authoritative Server → IP Address.
  4. Final Result: DNS returns IP to your browser.
  5. Connection Established: Browser connects to website.
