Simple HTML Guide – File Structure & Important Tags

This guide explains how to organize your website files and basic HTML concepts clearly for beginners.

What is the File Path System?

Key Idea: A website is made of multiple files (HTML, images, CSS, JS, etc.), organized in folders.

Why Do We Need File Paths?

When you want to show an image in your HTML file, the browser needs to know exactly where that image is.

Relative File Paths (Most Common)

Think of it like giving directions based on where you are now.

my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── hero.jpg
├── pages/
    ├── contact.html
    └── terms.html

Examples:

  • Same Folder: From index.html to about.html → <a href="about.html">About Us</a>
  • Sub-folder (Going down): From index.html to logo.png → <img src="images/logo.png" alt="Logo">
  • Parent folder (Going up): From contact.html to logo.png → <img src="../images/logo.png" alt="Logo">

Absolute File Paths

Full address from the web (use for external files only).

  • Example Link to Google: <a href="https://www.google.com" target="_blank">Search on Google</a>
  • Example External Image: <img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

Important Tip: Never use absolute local computer paths like C:/Users/Arjun/Desktop/image.png because they will break on other computers or when uploaded to the internet.

HTML Boilerplate (Basic Structure of Every Webpage)

Every webpage should start with this basic setup called a "boilerplate". It helps the browser understand the page properly. 

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My Website</title>
    </head>
    <body>
        
    </body>
</html>

Why is this important?

  • <!DOCTYPE html>: Tells browser to use the latest rules.
  • <meta charset="UTF-8">: Makes sure special characters (€, ₹) display properly.
  • <meta name="viewport">: Makes your site mobile-friendly.
  • <title>: Name shown in the browser tab.

The <div> Tag – The Generic Container

A <div> groups elements together so they can be treated as one block.

Example:


John Doe

Loves to code and teach HTML.

Why use a <div>?

  • To group related items together.
  • Make it easy to style them later (with CSS or JavaScript).

class vs id (Labels for Elements)

id: Unique label for ONE element (like a Social Security Number).

<div id="main-header">My Website</div>

class: Shared label for multiple elements (like a Job Title).

<div class="profile-card">User 1</div>
<div class="profile-card">User 2</div>

Use:

  • #id for unique elements.
  • .class for groups of similar elements.

The <span> Tag – Inline Highlighter

A <span> is like a highlighter pen that marks a word without breaking the line.

Example:

This is a very important message.

Why use a <span>?

  • Highlight part of a sentence.
  • Apply styles or JavaScript only to a word or phrase without creating new lines.
